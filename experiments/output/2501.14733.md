# Bridging the Gap: How LLMs Become HPC Experts with Real-Time Data Integration

High-Performance Computing (HPC) clusters are the workhorses of modern scientific discovery and complex simulations. Yet, despite their power, accessing and managing resources on these systems often feels like navigating a labyrinth, especially for users not steeped in command-line intricacies. Traditional graphical interfaces help, but mastering HPC still typically requires a deep understanding of specific commands and system quirks.

Recent advancements in Large Language Models (LLMs) offer a tantalizing possibility: what if an LLM could act as an expert guide, translating complex user requests into executable HPC commands and interpreting the results? While existing LLM-based tools can generate shell commands, they often fall short in integrating real-time, user-specific cluster information, which is crucial for tasks like checking job status or available resources. Fine-tuning LLMs on this dynamic data is impractical and costly.

This is where Retrieval-Augmented Generation (RAG) comes into play. RAG enhances LLMs by dynamically fetching relevant external information and incorporating it into the model's context. This is a better fit for the ever-changing state of an HPC cluster. Our paper introduces a novel extension to the standard RAG architecture specifically designed for HPC data, focusing on integrating both static documentation and dynamic, real-time command outputs.

The core innovation is **Hypothetical Command Embeddings (HyCE)**. Inspired by Hypothetical Document Embeddings (HyDE), HyCE helps the system accurately identify the correct HPC command to execute based on a user's natural language query. Instead of directly embedding the often cryptic command names, HyCE uses descriptive explanations of what commands do. When a user asks a question requiring real-time data (like "What GPUs are available?"), the system retrieves the relevant command *description*. Based on this description, the actual command (e.g., `nvidia-smi` or a cluster-specific script) is executed. The output of this command becomes a crucial piece of context for the LLM to formulate a precise, user-specific answer.

Figure 1 illustrates this extended RAG architecture. The user's query goes to a Retriever, which accesses both static HPC documentation ("HPC docs") and information about shell commands, enhanced by HyCE. For questions needing real-time data, HyCE facilitates identifying and executing the appropriate command. The resulting output ("get the Outputs") is then added to the context provided to the LLM, alongside relevant chunks from the documentation. The LLM can then synthesize an accurate, context-aware answer ("Answer") based on this combined information.

<br>![Figure 2: Example of comparing of two RAG responses. Unlike the ambiguous response provided by RAG with just HPC documents, RAG combined with HyCE can address user specific HPC questions more precisely.](../test/tem/2501.14733_Figure2.png)<br>

Figure 2 provides a concrete example. When asked "What GPUs are available for me?", a RAG system using only static documentation provides a generic, less helpful answer. In contrast, the RAG with HyCE executes a command to check available GPUs for that specific user and provides a precise list based on the command output. Similarly, for "What is the status of my job?", HyCE enables the system to run a command like `qstat` and report the actual job status, not just explain *how* to check it.

To evaluate this system, we developed an automated framework where the LLM itself generates synthetic, realistic Q&A pairs from HPC documentation and command outputs. The LLM also acts as a judge, evaluating the system's responses based on correctness and faithfulness. Our evaluation showed that incorporating HyCE significantly improved the RAG system's performance in answering HPC-specific queries, and these gains were further boosted by integrating other RAG improvement methods like Chain-of-Thought prompting and using stronger LLM/retrieval models.

Deploying LLMs in HPC environments raises critical security questions, particularly regarding data privacy and the risks of command execution. We address these through layered safeguards: training LLMs to be resilient to prompt injection, restricting command execution to only predefined, validated commands retrieved by HyCE, running the LLM under restricted user privileges, and containerizing the RAG application to isolate it from core system processes.

Our work demonstrates that by extending RAG with real-time command outputs via HyCE, LLMs can effectively transform into knowledgeable and helpful HPC expert assistants. This approach offers a scalable and adaptable way to make complex HPC systems more accessible to a broader user base, paving the way for more intuitive interfaces integrated into existing HPC portals.
