## Navigating the Online Learning Maze: A Multi-Agent RAG System to Boost Efficiency

Online learning has become indispensable, offering access to a wealth of knowledge across various platforms like YouTube, GitHub, and official documentation. Yet, piecing together information from these disparate sources can be a daunting task, often leading to cognitive overload and slowing down the learning process, especially in fast-paced technical fields. Imagine trying to learn a new programming framework – you might watch a video for an overview, dig through a GitHub repo for code examples, consult documentation for specifics, and then scour the web for troubleshooting tips. This manual cross-referencing is time-consuming and inefficient.

Addressing this challenge, researchers are exploring ways to automate and streamline this process. A recent poster paper introduces an early-stage Multi-Agent Retrieval-Augmented Generation (RAG) system designed to enhance online learning efficiency by integrating these diverse heterogeneous resources.

The core idea is to leverage the power of large language models (LLMs) like GPT-4o, but augment them with specialized agents capable of interacting with different types of online content. Instead of a single model trying to handle everything, the system employs a central Manager Agent that delegates user queries to a team of experts: a YouTube Video Agent, a GitHub Repository Agent, a Documentation Website Agent, and a Generic Search Engine Agent.

<br>![Figure 1: Working Flow of the Multi-Agent RAG System](../test/tem/2502.03948_Figure1.png)<br>

Each specialized agent uses resource-specific tools and APIs to retrieve relevant information. For instance, the YouTube agent might process video transcripts, the GitHub agent could pull code snippets or issue discussions, and the documentation agent would focus on relevant sections of official docs. This modular design allows for scalability and easy addition of new resource types or agents in the future.

The retrieved information from these agents is then processed. Relevant text is extracted from sources like transcripts or documentation pages. Embeddings are generated for this text using the LLM, capturing the semantic meaning. These embeddings, along with metadata like source URLs, are stored in a vector database (ChromaDB in this case), enabling efficient semantic search. For general web queries, the system retrieves information dynamically.

The Manager Agent takes the outputs from the specialized agents and synthesizes them into a cohesive, unified response tailored to the user's original query. The user interacts with the system through a simple interface, where they can specify the URLs of the resources they want to query and even refine the type of content to include (e.g., specific GitHub content like code or issues). This allows users to focus their questions on preselected sources, potentially improving relevance and reducing irrelevant results.

A preliminary user study was conducted with 15 master's-level computer science students, chosen for their familiarity with navigating online resources for learning. The study used the Technology Acceptance Model (TAM) to assess the system's usability and utility. The TAM questionnaire focuses on two key constructs: Perceived Usefulness (PU) and Perceived Ease of Use (PEU).

<br>![Figure 2: Analysis of the user study using TAM](../test/tem/2502.03948_Figure2.png)<br>

The results showed a strong perceived ease of use, with participants generally agreeing on the system's usability. The perceived usefulness scores were moderate-high on average but showed more variability among participants. This suggests that while many found the system useful for their learning tasks, there's potential to better align the system's utility with diverse user needs and expectations. Overall TAM scores were favorable, reflecting the combined positive influence of usability and moderate utility.

This early work demonstrates the potential of a multi-agent RAG system to make online learning more efficient by automating the tedious task of integrating knowledge from scattered resources. Future work aims to make the retrieval system more dynamic, allowing it to intelligently decide which agents to use for optimal performance. The researchers also plan to test the system with a wider range of users, integrate more domain-specific platforms, and explore personalization techniques to further enhance the learning experience. Such a system holds promise for anyone who relies on the vast, yet fragmented, landscape of online information for skill acquisition and problem-solving.
